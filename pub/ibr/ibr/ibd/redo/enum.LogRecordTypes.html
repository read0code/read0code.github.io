if the mtr contains only one log record for one page, i.e., write_initial_log_record has been called only once, this flag is ORed to the type of that first log record

MLOG_INIT_FILE_PAGE = 29

this means that a file page is taken into use and the prior contents of the page should be ignored: in recovery we must not trust the lsn values stored to the file page. Note: it’s deprecated because it causes crash recovery problem in bulk create index, and actually we don’t need to reset page lsn in recv_recover_page_func() now.
